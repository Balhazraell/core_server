Перед тем как отдать веб-страницу убедимся что у нас собрана самая актуальная версия клиентского 
приложения. Для этого на компьютере должна быть последняя версия node и webpack.
Для удобства лучше всего установить и nvm.
1) В оффицальном github (https://github.com/creationix/nvm) есть полня инструкция.
2) Через него устанавливаем последнюю версию node:
2.1) nvm ls-remote - покажет все версии (для windows по другому)
2.2) nvm install v10.9.0 64 - устанавливаем версию и 32/64 битную.
2.3) nvm use v10.9.0 64 - выбираем версию которую хотим использовать.
! На Windows будут проблемы если путь в учетную запись имеет кириллицу...!

// Устарело.
# 3) Устанавливаем webpack глобально(начиная с какой-то там версии по мимо самого webpack нужно ставить еще и 
# webpack-cli). Так что если вылезет сообщение что надо ещё что-то зоставить - не пугаемся и ставим.
# 3.1) npm -i -g webpack
# 3.2) npm -i -g webpack-cli

3) Переходим в папку static/js/ и выполняем команду npm install

Для сборки проекта заходим в папку static/js/, так как именно тут лежит файл с инструкцией webpack.config.js
и выполняем команду:
```
~$ npx webpack
```
произодет сборка клиентской части приложения.

В серверной части используется библиотека golang.org/x/net/websocket для корректной работы её надо установить локально:
go get golang.org/x/net/websocket

Установкеа RabbitMQ и его настройка.
1) Устанавливаем RabbitMQ.
2) Устанавливаем плагин работы с визуальной оболочкой RabbitMQ - rabbitmq-plugins enable rabbitmq_management.
RabbitMQ имеет несколько консольных приложений для работы:
* rabbitmq-plugins - установка плагинов для RabbitMQ
* rabbitmqctl - основная консольная утилита для администрирования RabbitMQ.
* rabbitmqadmin - Оператор тасков для HTTP API ??? (ну или что-то такое)

 3) Создаем роль:
 rabbitmqctl add_user name password
 4) Создаем хост:
 rabbitmqctl add_vhost host_name
 5) Назначем хосту пользователя:
 rabbitmqctl set_permissions -p host_name user_name .* .* .*

 (Все это можно сделать через визуальную оболочку)
